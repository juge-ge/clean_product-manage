## 1. 模块概述

### 1.1 模块定位
清洁生产政策与管理模块是清洁生产管理系统的核心业务模块，负责提供政策信息、动态资讯、法规政策、能力建设和会议宣传等综合服务，为用户提供全面的清洁生产政策支持。

### 1.2 模块结构
```
清洁生产政策与管理
├── 通知公告 (已实现)
├── 动态信息
│   ├── 国外动态
│   ├── 国内动态
│   └── 省内动态
├── 清洁生产法规政策
│   ├── 国家政策法规
│   ├── 各省政策法规
│   ├── 清洁生产审核指南
│   ├── 清洁生产评价指标体系
│   ├── 清洁生产技术导向目录
│   ├── 淘汰落后生产能力、工艺和产品目录
│   └── 国家推荐的清洁生产成熟技术
├── 能力建设
│   ├── 咨询人员
│   └── 管理人员
└── 会议与宣传
    ├── 清洁生产会议
    └── 清洁生产宣传
```

### 1.3 技术特点
- **前后端解耦**: 采用Vue.js + API接口的架构模式
- **响应式设计**: 支持多设备适配
- **模块化开发**: 各子模块独立开发，便于维护
- **统一风格**: 遵循现有系统设计规范

## 2. 技术架构设计

### 2.1 前端技术栈
- **框架**: Vue 3.x + Composition API
- **UI组件库**: Naive UI (与现有系统保持一致)
- **状态管理**: Pinia
- **路由管理**: Vue Router 4.x
- **HTTP客户端**: Axios
- **样式**: SCSS + CSS Variables

## 3. 动态信息模块

### 3.1 功能概述
动态信息模块提供国内外清洁生产相关的最新资讯，分为国外动态、国内动态、省内动态三个子模块，实现信息的分类展示和统一搜索。

### 3.2 页面设计
参考现有通知公告模块的设计模式，采用三栏布局：
- **首页模式**: 三栏展示，每栏显示最新3条动态
- **列表模式**: 单栏展示，支持分页和搜索
- **详情模式**: 弹窗展示详细信息

### 3.3 数据结构
```javascript
// 动态信息数据结构
const dynamicInfo = {
  id: Number,           // 唯一标识
  title: String,        // 标题
  content: String,      // 内容
  summary: String,      // 摘要
  publishDate: String,  // 发布日期
  source: String,       // 来源
  type: String,         // 类型: 'domestic' | 'provincial' | 'international'
  province: String,     // 省份 (省内动态)
  country: String,      // 国家 (国外动态)
  tags: Array,          // 标签
  viewCount: Number,    // 浏览次数
  isTop: Boolean,       // 是否置顶
  attachments: Array    // 附件
}
```

### 3.4 组件设计

### 3.5 交互逻辑
- **搜索功能**: 支持标题、内容、来源关键词搜索
- **分类筛选**: 按动态类型筛选
- **分页展示**: 每页10条，支持快速跳转
- **详情查看**: 点击标题查看完整内容

## 4. 清洁生产法规政策模块

### 4.1 功能概述

## 5. 能力建设模块

- **双栏布局**: 咨询人员和管理人员分别展示

## 7. API接口设计

### 7.1 动态信息API
```javascript
// web/src/api/policy.js
export const policyApi = {
  // 动态信息相关
  getDynamicList: (params) => request.get('/api/v1/policy/dynamic', { params }),
  getDynamicDetail: (id) => request.get(`/api/v1/policy/dynamic/${id}`),
  searchDynamics: (params) => request.get('/api/v1/policy/dynamic/search', { params }),
  
  // 法规政策相关
  getRegulationList: (params) => request.get('/api/v1/policy/regulation', { params }),
  getRegulationDetail: (id) => request.get(`/api/v1/policy/regulation/${id}`),
  getRegulationByCategory: (category, params) => request.get(`/api/v1/policy/regulation/category/${category}`, { params }),
  downloadRegulation: (id) => request.get(`/api/v1/policy/regulation/${id}/download`, { responseType: 'blob' }),
  
  // 能力建设相关
  getPersonnelList: (params) => request.get('/api/v1/policy/personnel', { params }),
  getPersonnelDetail: (id) => request.get(`/api/v1/policy/personnel/${id}`),
  getPersonnelByType: (type, params) => request.get(`/api/v1/policy/personnel/type/${type}`, { params }),
  
  // 会议与宣传相关
  getMeetingList: (params) => request.get('/api/v1/policy/meeting', { params }),
  getMeetingDetail: (id) => request.get(`/api/v1/policy/meeting/${id}`),
  getVideoList: (params) => request.get('/api/v1/policy/video', { params }),
  getVideoDetail: (id) => request.get(`/api/v1/policy/video/${id}`),
  updateVideoViewCount: (id) => request.post(`/api/v1/policy/video/${id}/view`)
}
```

## 总结

本技术文档详细规划了清洁生产政策与管理模块的构建方案，包括：

1. **模块化设计**: 四个子模块独立开发，便于维护和扩展
2. **前后端解耦**: 通过API接口实现数据交互，支持Mock数据开发
3. **统一设计规范**: 遵循现有系统风格，确保用户体验一致
4. **响应式布局**: 支持多设备适配，提升用户体验
5. **组件化开发**: 通用组件复用，提高开发效率

该方案确保了模块的可维护性、可扩展性和用户体验，为清洁生产管理系统的完善提供了坚实的技术基础。
